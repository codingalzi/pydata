머신러닝 맛보기 1편¶

참고: 오렐리앙 제롱의 <핸즈온 머신러닝(2판)> 1장의 소스코드를 사용합니다.

주요 내용¶

머신러닝은 특정 값을 예측하는 모델(일종의 함수)을 구현하는 컴퓨터 프로그래밍 분야이며 주어진 문제에 대한 적절한 모델을 찾는 일이 핵심 과제이다. 머신러닝은 현재 데이터 분석의 주요 활용분야가 자리잡았으며 지금까지 살펴 본 다양한 개념과 도구들이 유용하게 활용된다. 여기서는 머신러닝의 가장 기본 모델인 선형회귀 모델을 직접 구현하면서 머신러닝의 기본 아이디어를 전달한다. 또한 모든 머신러닝 알고리즘의 핵심인 경사하강법도 함께 설명한다.

선형회귀 모델을 구현하는 과정을 통해 다섯 단계로 이루어진 머신러닝 문제해결의 전형적인 과정을 살펴본다.

  • 문제 정의
  • 데이터 구하기
  • 데이터 적재와 준비
  • (선형회귀) 모델 훈련/학습
  • 학습 모델 활용

1단계: 문제 정의¶

OECD 국가들을 대상으로 1인당 GDP와 삶의 만족도 사이의 관계를 선형회귀 모델로 구현하려 한다.

참고: 국가별 명목 GDP 순서에서 2020년 기준 대한민국은 세계 10위이다.

2단계: 데이터 구하기¶

국가별 '삶의 만족도'(life satisfaction)는 OECD가 매년 발표하는 '더 나은 삶의 지수'(BLI, Better Life Index) 자료에 포함된다. 반면에 1인당 GDP 자료는 국제통화기금(IMF)의 세계 경제전망 데이터베이스에서 제공한다.

여기서는 2015년 기준 자료를 아래 링크에서 미리 준비된 두 개의 데이터 파일을 다운로드하는 것으로 대체한다. 다만 실제 자료를 아래와 같이 잘 정리된 데이터 파일로 만드는 과정이 일반적으로 간단하지 않다는 점만을 언급해둔다.

  • oecd_bli_2015.csv: 2015년도 기준 OECD 국가들의 '더 나은 삶의 지수'(BLI) 데이터
  • gdp_per_capita.csv: IMF(국제통화기금)에 제공하는 1인당 국내총생산 데이터(GDP) 데이터

다운로드 경로는 다음과 같다.

In [4]:
datapath = "https://raw.githubusercontent.com/codingalzi/handson-ml2/master/notebooks/datasets/lifesat/"

위 경로에 두 파일의 이름을 각각 추가하면 해당 파일들을 다운로드하여 컴퓨터에 저장할 수 있다. 하지만 여기서는 저장하는 대신에 이어서 설명하는 데이터 적재과정을 동시에 진행하는 것으로 데이터 구하기 단계를 해결한다.

3단계: 데이터 적재와 전처리¶

일반적으로 저장된 데이터를 바로 모델 훈련(학습)에 사용할 수 없으며, 적절한 내용을 담은 데이터셋 자료형으로 변환해야 한다. 인터넷의 어떤 서버에 존재하거나 컴퓨터에 저장되어 있는 파일을 특정 자료형의 값으로 불러오는 과정을 데이터 적재(data loading)이라 부른다. 반면에 적재된 데이터를 적절한 내용을 포함하고 적절한 형식을 따르는 값으로 변환하는 과정은 데이터 전처리(data preprocessing)라 한다.

삶의 만족도 데이터 적재와 전처리¶

OECD 국가별 삶의 만족도 데이터는 더 나은 삶의 지수(BLI) 데이터 파일에 포함되어 있다. 따라서 먼저 해당 csv 파일을 판다스의 데이터프레임 객체(값)으로 불러온 후에 삶의 만족도와 관련된 내용을 추출하는 과정을 밟는다.

따라서 'Indicator' 의 항목에 대한 각 국가별 수치만을 추출하자. 이를 위해 국가명('Country' 열의 항목)을 행의 인덱스 이름으로, 'Indicator'의 항목을 열의 인덱스 이름으로 사용하는 데이터프레임을 생성한다. 해당 열과 항목에 해당하는 값은 'Values' 열에 포함된 값을 사용한다.

이것이 가능하려면 전제조건은 새로운 데이터프레임의 항목값을 유일하게 지정할 수 있어야 한다. 그런데 OECD의 원본 파일에는 각 측정지표의 값을 국가별로 소득 불평등('INEQUALITY')과 관련된 5가지 기준에 따라 다른 값이 포함된다. 사용된 기준은 다음과 같다.

  • 'TOT': 인구 전체 대상
  • 'MN': 남성 대상
  • 'WMN': 여성 대상
  • 'HGH': 소득이 높은 사람 대상
  • 'LW': 소득이 낮은 사람 대상

데이터 병합¶

두 데이터를 하나의 데이터프레임으로 병합한다. 삶의 지수 데이터를 왼편에, 1인당 GDP 데이터를 오른편에 위치시키며 기준은 국가명이다.

참고: 두 데이터 파일을 병합할 때 공통 인덱스에 해당하는 행(row)만 사용되도록 설정된다.

In [13]:
full_country_stats = pd.merge(left=oecd_bli, right=gdp_per_capita, left_index=True, right_index=True)

또한 1인당 GDP 기준으로 오름차순으로 국가를 정렬시킨다.

In [14]:
full_country_stats.sort_values(by="GDP per capita", inplace=True) # 1인당 GDP 기준으로 정렬

4단계: (선형회귀) 모델 훈련/학습¶

선형 관계 확인¶

위 산점도에 따르면 1인당 GDP와 삶의 만족도가 어느 정도 선형 관계에 있는 것처럼 보인다. 아래 도표에서 그려진 세 개의 직선 중에서 파랑색 실선이 선형 관계를 가장 적절하게 나타낸다.

In [20]:
import numpy as np

선형회귀 모델 훈련/학습¶

선형 관계는 위 도표에서처럼 직선으로 구현되는 선형회귀 모델(linear regression model)은 직선의 절편(Y축과 만나는 점)과 기울기 두 개의 모델 파라미터(model parameters)에 의해 결정된다. 예를 들어, 1인당 GDP와 삶의 만족도는 아래 선형 모델로 표현할 수 있다.

$$ \text{삶의만족도} = \theta_0 + \theta_1 \times \text{1인당GDP} $$

아래 코드는 최적의 선형 모델의 절편과 기울기를 알아내는 모델의 생성과 훈련과정을 나타낸다.

  • 모델 지정: 사이킷런(sklearn) 패키지의 linear_model 모듈에 포함된 LinearRegression 클래스 활용
  • 훈련 데이터넷 지정
    • 특성 데이터: 1인당 GDP
    • 타깃 데이터: 삶의 만족도
  • 모델 훈련: fit() 메서드 호출
    • 반환값: 훈련된 선형회귀 모델
In [21]:
from sklearn import linear_model

# 선형회귀 모델 지정

# 훈련 데이터셋 지정

ysample = np.c_[sample_data["Life satisfaction"]]

# 모델 훈련
lin1.fit(Xsample, ysample)

  • coef_[0]: 직선의 기울기

과대적합¶

훈련된 모델이 훈련데이터에는 너무나도 잘 맞지만 새로운 데이터에 대해서는 잘 작동하지 않을 때 해당 모델이 훈련데이터에 과대적합되었다고 말한다.

예를 들어, 아래 코드는 고차항 다항회귀를 이용하여 선형이 아니라 60차 다항식의 그래프로 모델을 예측한다. 결과는 훈련데이터셋에 매우 잘 들어맞지만, 새루운 데이터에 대해 훈련된 모델이 제대로된 예측을 할 수 있을 거라고 믿기는 어렵다. 이런 모델은 일반화가 어렵다라고 말한다.
